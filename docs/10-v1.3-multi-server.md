# 10 — Sprint v1.3: Multi Server Support

**Target:** Satu dashboard untuk monitor banyak server
**Estimasi:** 2 minggu
**Dependensi:** v1.2 selesai

**Catatan:** Arsitektur v1.0 sudah mendukung multi-server (`server_id` di semua tabel). Sprint ini fokus pada **UX multi-server**, **SSL monitor**, **port scanner**, **env checker**, **Docker**, **Python**, **Network monitor** — modul yang belum tersentuh di v1.0-v1.2.

## Week 2: Multi-Server UX & Polish

### 2.1 Multi-Server Dashboard
- [ ] Overview: grid server card, masing-masing summary health + alert count
- [ ] Server selector dropdown di topbar → filter seluruh dashboard
- [ ] "All Servers" view — aggregate metric, total alert, services health matrix
- [ ] Server grouping by tag: production, staging, development

### 2.2 Cross-Server Operations
- [ ] Bulk service restart (restart nginx di semua server production)
- [ ] Bulk alert acknowledge
- [ ] Compare metrics antar server (overlay chart)

### 2.3 Server Health Score

```php
// Health score 0-100 per server
// Komponen:
// - CPU usage (25%)
// - RAM usage (25%)
// - Disk usage (20%)
// - Critical services running (30%) — 1/N per service
// - Active critical alerts (-10 per alert)
```

- [ ] `ServerController@summary` — return health score + breakdown
- [ ] UI: health score badge di server card (hijau >80, kuning 50-80, merah <50)

### 2.4 Scheduler Update
- [ ] Semua job support multi-server: loop all active servers, dispatch per-server job
- [ ] `CheckSslJob` — setiap 5 menit
- [ ] `CheckEnvJob` — setiap 10 menit
- [ ] `ScanPortJob` — manual trigger / harian

- [ ] Port scanner: scan TCP/UDP port, lihat status open/closed

- [ ] Docker: list container, restart/start/stop, lihat logs
- [ ] Python: list python process + resource usage

- [ ] Semua test pass
