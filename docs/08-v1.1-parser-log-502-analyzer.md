
## Week 2: 502 Error Analyzer

### 2.1 Analysis Engine

```php
class Error502Analyzer
{
    // Trigger: on-demand via API, atau auto saat Nginx error 502 terdeteksi
    public function analyze(Server $server, ?string $domain = null): Error502Analysis
    {
        // 1. Cek Nginx error log — cari "502 Bad Gateway"
        $nginxErrors = LogEntry::where('server_id', $server->id)
            ->where('source', 'nginx-error')
            ->where('level', 'error')
            ->where('message', 'like', '%502%')
            ->recent()
            ->get();

        // 2. Cek PHP-FPM log — cari "max_children", "busy", "segfault"
        $phpFpmErrors = LogEntry::where('server_id', $server->id)
            ->where('source', 'php-fpm')
            ->where('level', 'error')
            ->recent()
            ->get();

        // 3. Cek metric snapshot saat 502 terjadi
        $metrics = Metric::where('server_id', $server->id)
            ->whereIn('type', ['cpu', 'ram'])
            ->where('recorded_at', '>=', $timeWindow)
            ->get();

        // 4. Cek service status history
        $serviceStatus = ServiceStatus::whereIn('service_id', ...)
            ->where('checked_at', '>=', $timeWindow)
            ->get();

        // 5. Root cause analysis
        return app(RootCauseEngine::class)->determine(
            nginxErrors: $nginxErrors,
            phpFpmErrors: $phpFpmErrors,
            metrics: $metrics,
            serviceStatus: $serviceStatus,
        );
    }
}
```

### 2.2 Root Cause Engine

| Evidence | Probable Cause | Confidence | Recommendation |
|----------|---------------|------------|----------------|
| PHP-FPM "max_children reached" + CPU spike | PHP-FPM process pool exhausted | 90% | Increase pm.max_children, check slow queries |
| PHP-FPM status = down + Nginx "connect() failed" | PHP-FPM service crash | 85% | Restart PHP-FPM, check error log |
| Nginx "upstream timed out" + RAM >95% | OOM killing backend | 80% | Increase RAM / reduce workers |
| PHP-FPM "segfault" + specific path | Bug in PHP code at that path | 75% | Check recent deploy, rollback if needed |
| Nginx "connect() to unix:...php-fpm.sock failed" + socket file missing | PHP-FPM socket not created | 90% | Check PHP-FPM listen config, restart PHP-FPM |
| MySQL slow_queries spike + PHP-FPM busy | Database bottleneck causing 502 cascade | 70% | Optimize queries, increase connection pool |
| Nginx error "104: Connection reset by peer" + PHP-FPM restart log | PHP-FPM was restarted during request | 85% | Graceful restart: `kill -USR2` instead of restart |
